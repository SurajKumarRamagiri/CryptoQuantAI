# DriftForge Design Rationale

## Visual Direction
- Domain-driven terminal aesthetic for a trading simulator and research platform.
- Primary palette avoids generic purple-fintech defaults.
- Typography pairing uses strong display hierarchy and readable body text.

## Token Rules
- All key colors come from frontend token file.
- Components should avoid ad-hoc hardcoded colors.

## Anti-Vibecoded Guardrails
- No purple-gradient-first identity.
- No generic glassmorphism-heavy baseline.
- Avoid boilerplate SaaS hero copy unrelated to trading simulation.

## Mobile-First Rules
- 44x44 minimum touch targets.
- No horizontal overflow on supported breakpoints.
- Core journey parity required for login, trade, portfolio, and analysis.
